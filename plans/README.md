## Repo-wide facts every executor must know

- **PHP 8.4 + MySQL 8.0**, custom framework (not Laravel/Symfony). Do **not** upgrade
  syntax or dependencies unless the plan says to.
- **`manager/` and `site/` are two environments sharing identical framework code.**
  `app/inc/lib/*` and `app/inc/model/*` are **byte-for-byte identical** between the two
  trees (verified with `diff -rq`). Any change to a shared file **must be applied to both
  copies**, or they silently drift. Each plan that touches shared code lists both paths.
- **`kernel.php` is gitignored.** Tests and the app need it. Copy from the example first:
  `cp site/app/inc/kernel.php.example site/app/inc/kernel.php` (and the manager one).
- **Verification commands**:
  - PHPStan (level 3, no DB needed — the reliable host-side gate):
    `cd manager && php app/inc/lib/vendor/bin/phpstan analyse` (repeat for `site`)
  - PHPUnit (**needs a live MySQL + kernel.php**; run inside Docker):
    `docker exec leggo php /var/www/leggo/manager/app/inc/lib/vendor/bin/phpunit`
    (repeat for `site`). On the host, PHPUnit currently fails to connect (see `TODOS.md`),
    so treat the Docker container as the test environment.
  - Single test: append `--filter testMethodName`.
- **Commits**: Conventional Commits in PT-BR (`feat:`, `fix:`, `chore:`) — match
  `git log`. Example: `fix: remove runner de migrations exposto no web root`.
- Do **not** push or open PRs unless the operator says so.

## Findings considered and rejected

- *Manager/site code duplication is "wrong"*: it is an intentional whitelabel two-env
  structure (AGENTS.md: "Same code structure, different kernel.php"). Plan 008 reduces the
  duplication's maintenance cost but is **optional** and MED-risk — not a defect to fix
  blindly. Do not collapse the two environments into one.
- *`unsafe-eval` in CSP*: required by Alpine.js expression evaluation (`@click`, `x-data`
  in the UI). Plan 007 keeps `unsafe-eval` and targets only the lower-hanging hardening.
- *`localPDO` global-transaction-per-request with `__destruct` rollback* (flagged by the
  second-pass framework audit as "silent data loss if a controller renders without redirecting"):
  this is **documented, intentional design** — AGENTS.md states `basic_redir()` is the commit
  gate and `__destruct` does a safety rollback when no explicit redirect occurred. Not a defect;
  do not "fix" it. (The risk is real if a future controller renders-without-redirecting after a
  write — a convention to enforce in review, not a code change.)
- *`save()` INSERT-vs-UPDATE inferred by string-matching the filter to `"active = 'yes'"`*
  (`DOLModel.php`): genuinely brittle (whitespace/case/quote variance flips intent), but
  **deferred, not rejected** — lower leverage than the P1 security set and needs a careful
  API change (explicit `insert()`/`update()`). Spec it as a follow-up if the operator wants it.
- *`rootOBJ::__call` magic `set_*`/`get_*`*: defeats static analysis and silently returns
  `null` on a typo'd method. Real tech-debt, **deferred** — replacing it with the already-
  documented typed methods is a medium refactor better done after the P1 set lands.
- *Manager/site duplication via a shared tree* — still covered (and still optional/MED-risk) by
  **plan 008**. The second-pass audit re-confirmed the duplication's cost but did not change the
  recommendation: reduce maintenance cost, do not collapse the two environments into one.
